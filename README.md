# KCD2 Dice — браузерная игра в кости на двоих по LAN

Веб-реализация игры в кости из Kingdom Come: Deliverance II. Два игрока, локальная сеть или Radmin VPN, игра в браузере без установки.

## Правила

6 кубиков. На своём ходу: бросаешь — выбираешь зачётные кости — решаешь продолжить (с риском зонка) или зафиксировать очки.

**Комбинации:**

| Комбинация | Очки |
|---|---|
| Одиночная «1» | 100 |
| Одиночная «5» | 50 |
| Три «1» | 1000 |
| Три N (N=2..6) | N × 100 |
| 4 одинаковых | ×2 от тройки |
| 5 одинаковых | ×4 от тройки |
| 6 одинаковых | ×8 от тройки |
| Стрит 1-5 | 500 |
| Стрит 2-6 | 750 |
| Стрит 1-6 | 1500 |

- **Зонк:** в броске нет ни одной зачётной → все очки хода сгорают, ход переходит.
- **Hot dice:** все 6 костей зачётные → обязательный переролл всех 6.
- **Стриты** засчитываются только в рамках одного броска.
- **Победа:** первый, у кого после фиксации очки ≥ цели (2000 / 3000 / 4000 — выбирается перед партией).

## Запуск

Требуется Node.js 20+ ([скачать](https://nodejs.org/)).

```bash
# в папке проекта
npm install
npm test            # юнит-тесты движка
npm start           # сервер на http://localhost:8080
npm start -- --port 9000   # другой порт
```

Откройте `http://localhost:8080` в браузере. Хост — окно №1. Для второго игрока — см. ниже.

## Игра против бота (v0.6+)

Если играть некому — добавьте AI-противника прямо из лобби.

1. Запустите сервер (`npm start`), откройте `http://localhost:8080`, введите имя.
2. В свободном слоте лобби нажмите кнопку **«+ Добавить бота»**.
3. Бот появится со случайным именем (например, «Шумахер», «Индро», «Митяй»).
4. Выберите цель партии и нажмите «Начать игру».
5. После окончания партии в финальной модалке откроется **полное имя бота** + описание стиля игры — это и есть его «настроение», которое он скрывал всю партию.

**Стили (настроения) бота** — выбираются случайно для каждой партии:

| Прозвище | Поведение |
|---|---|
| Осторожный | Банкует рано, минимум риска |
| Расчётливый | Сбалансированная стратегия |
| Дерзкий | Идёт на лишний бросок чаще, охотится за hot dice |
| Безбашенный | Банкует только под победу или огромный замес |
| Хладнокровный | Чистая логика, без эмоций и рофла |
| Шальной | Непредсказуем — порог риска случайный на каждое решение |

Бот **рискует больше**, когда соперник близок к победе или сам отстаёт. У большинства настроений ~5 % шанс «рофла» — случайной инверсии решения. Чтобы убрать бота (если зашёл живой второй игрок), нажмите «× убрать» рядом с именем бота в лобби.

## Игра с товарищем по Radmin VPN

Кратко: оба ставите [Radmin VPN](https://www.radmin-vpn.com/) и заходите в одну виртуальную сеть. Хост поднимает `npm start`, второй игрок подключается к серверу по Radmin-IP хоста.

### Шаги для хоста (создаёт сеть и поднимает сервер)

1. Скачайте и установите [Radmin VPN](https://www.radmin-vpn.com/) (Windows / macOS / Linux).
2. Запустите Radmin VPN. В верхнем меню: **Network → Create network**.
3. В диалоге задайте:
   - **Network name** — придумайте уникальное имя (например, `kcd2-dice-sasha`).
   - **Network password** — пароль для входа.
   - Нажмите **Create**.
4. После создания сеть появится в списке. Рядом с **вашим именем** будет показан ваш Radmin-IP (формат `26.x.x.x` или `25.x.x.x`).
5. Передайте товарищу **имя сети и пароль** (через Telegram / Discord / SMS — не публикуйте).
6. В терминале запустите сервер: `npm start`. Узнайте свой IP — см. ниже.
7. Откройте `http://localhost:8080` в браузере. Введите имя.

### Шаги для второго игрока (подключается к сети хоста)

1. Скачайте и установите [Radmin VPN](https://www.radmin-vpn.com/).
2. Запустите Radmin VPN. В верхнем меню: **Network → Join network**.
3. В диалоге введите:
   - **Network name** — имя сети, которое прислал хост.
   - **Network password** — пароль, который прислал хост.
   - Нажмите **Join**.
4. После присоединения в окне Radmin VPN вы увидите **список участников сети**. Хост будет в списке с его именем (которое он указал в Radmin) и зелёной точкой «онлайн».
5. **Скопируйте Radmin-IP хоста:**
   - В списке участников рядом с именем хоста показан его IP (формат `26.x.x.x` или `25.x.x.x`).
   - Кликните по имени хоста правой кнопкой → **Copy IP-address**. Или просто запомните/выпишите цифры.
6. Откройте браузер и перейдите по адресу `http://<ip-хоста>:8080`. Пример: `http://26.155.42.18:8080`.
7. Введите имя — попадёте в лобби. Когда оба видны со статусом «онлайн», хост нажимает «Начать игру».

### Как ещё узнать свой Radmin-IP (если не виден сразу)

- **Способ 1 (рекомендуется):** В окне Radmin VPN рядом с именем сети сверху показан ваш IP в этой сети.
- **Способ 2:** Кликните правой кнопкой по своему имени в списке участников → **Copy IP-address**.
- **Способ 3 (через командную строку):**
  ```
  ipconfig
  ```
  Найдите блок **Ethernet adapter Radmin VPN** → строка `IPv4-адрес` / `IPv4 Address`. Это и есть ваш Radmin-IP.

### Если не подключается

- **Брандмауэр Windows:** при первом `npm start` Windows может спросить «разрешить Node.js доступ к сети» — поставьте обе галки (частная + публичная сеть). Если уже отказали — откройте «Брандмауэр Windows → Разрешение взаимодействия с приложением» и разрешите `node.exe`. Или временно отключите фаервол для проверки.
- **Не та сеть:** оба должны быть в **одной и той же** Radmin-сети (имя совпадает, цвет иконки одинаковый).
- **Проверка связи:** во второй машине в командной строке: `ping <ip-хоста>` — должны идти ответы. Если пакеты теряются — проблема в Radmin/фаерволе.
- **Антивирус:** некоторые антивирусы блокируют локальные серверы. Добавьте `node.exe` в исключения.
- **Порт занят:** если `npm start` ругается на занятый порт — запустите на другом: `npm start -- --port 9090`, тогда второй игрок открывает `http://<ip-хоста>:9090`.

## Управление

| Действие | Кнопка / Hotkey |
|---|---|
| Выбрать/снять кость | клик по кости |
| Бросить кости (первый бросок хода) | кнопка «Бросить» / `Space` |
| Записать выбор и перебросить оставшиеся | кнопка «Записать и продолжить» / `F` |
| Записать выбор и закончить ход (бэнк) | кнопка «Записать и закончить ход» / `Q` |
| Помощь (правила) | `T` |

## Технологии

- **Сервер:** Node.js 20+, Express, `ws` (нативный WebSocket).
- **Клиент:** vanilla HTML / CSS / ES-модули, без бандлера и без фреймворков.
- **Тесты:** встроенный `node:test`.
- **Авторитативный сервер:** клиент шлёт только намерения, сервер бросает кубики через `crypto.randomInt` и валидирует все ходы. Анти-чит «из коробки».

## Структура

```
server/
  index.js          # Express static + WebSocket upgrade
  room.js           # GameRoom: FSM ходов, broadcast
  protocol.js       # типы сообщений + валидация
  engine/
    rules.js        # константы
    scoring.js      # scoreSelection(dice) → {score, valid, breakdown}
    validate.js     # canHold, canBank, checkWin, isFarkle
public/
  index.html        # лобби + игровой экран + модалки
  styles/           # main.css (layout) + dice.css (грани + tumble)
  js/
    main.js         # bootstrap, hotkeys
    net.js          # WebSocket + reconnect backoff
    state.js        # snapshot store + selection
    ui.js           # рендер DOM по снапшоту
    engine/         # зеркало серверного движка (preview-скор)
tests/
  scoring.test.js   # 40+ кейсов
  validate.test.js
```

## Не реализовано (есть в KCD2)

- **Специальные кости** (например, «Чёртова голова» — джокер). Используются только стандартные кости 1..6.
- **Бляхи** (одноразовые предметы вроде «Серебряная бляха удвоения»).
- **Подбор разных костей перед игрой.**
- **Ставки и валюта.**

Эти фичи могут быть добавлены позже — движок и протокол к этому готовы.
