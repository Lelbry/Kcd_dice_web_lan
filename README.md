# KCD2 Dice — браузерная игра в кости на двоих по LAN

Веб-реализация игры в кости из Kingdom Come: Deliverance II. Два игрока, локальная сеть или Radmin VPN, игра в браузере без установки.

## Правила

6 кубиков. На своём ходу: бросаешь — выбираешь зачётные кости — решаешь продолжить (с риском зонка) или зафиксировать очки.

**Комбинации:**

| Комбинация | Очки |
|---|---|
| Одиночная «1» | 100 |
| Одиночная «5» | 50 |
| Три «1» | 1000 |
| Три N (N=2..6) | N × 100 |
| 4 одинаковых | ×2 от тройки |
| 5 одинаковых | ×4 от тройки |
| 6 одинаковых | ×8 от тройки |
| Стрит 1-5 | 500 |
| Стрит 2-6 | 750 |
| Стрит 1-6 | 1500 |

- **Зонк:** в броске нет ни одной зачётной → все очки хода сгорают, ход переходит.
- **Hot dice:** все 6 костей зачётные → обязательный переролл всех 6.
- **Стриты** засчитываются только в рамках одного броска.
- **Победа:** первый, у кого после фиксации очки ≥ цели (2000 / 3000 / 4000 — выбирается перед партией).

## Запуск

Требуется Node.js 20+ ([скачать](https://nodejs.org/)).

```bash
# в папке проекта
npm install
npm test            # юнит-тесты движка
npm start           # сервер на http://localhost:8080
npm start -- --port 9000   # другой порт
```

Откройте `http://localhost:8080` в браузере. Хост — окно №1. Для второго игрока — см. ниже.

## Игра с товарищем по Radmin VPN

1. На обеих машинах: установите [Radmin VPN](https://www.radmin-vpn.com/), создайте/присоединитесь к одной сети.
2. **Хост:** `npm start`. Узнайте свой Radmin-IP в окне Radmin (вкладка «Network», рядом с именем — это ваш виртуальный IP, например `26.x.x.x`).
3. **Хост:** откройте `http://localhost:8080`. Введите имя, нажмите «Сохранить».
4. **Второй игрок:** откройте `http://<radmin-ip-хоста>:8080`. Введите имя.
5. Когда оба в лобби (по точкам онлайн), хост выбирает цель и нажимает «Начать игру».

Если второй игрок не может подключиться:
- Проверьте, что Windows Firewall не блокирует порт 8080. Разрешите Node.js доступ к сети, или временно отключите фаервол для теста.
- Убедитесь, что оба точно в одной Radmin-сети (одинаковый цвет/название).
- Попробуйте `ping <radmin-ip>` со второй машины.

## Управление

| Действие | Кнопка / Hotkey |
|---|---|
| Выбрать/снять кость | клик по кости |
| Бросить кости (первый бросок хода) | кнопка «Бросить» / `Space` |
| Записать выбор и перебросить оставшиеся | кнопка «Записать и продолжить» / `F` |
| Записать выбор и закончить ход (бэнк) | кнопка «Записать и закончить ход» / `Q` |
| Помощь (правила) | `T` |

## Технологии

- **Сервер:** Node.js 20+, Express, `ws` (нативный WebSocket).
- **Клиент:** vanilla HTML / CSS / ES-модули, без бандлера и без фреймворков.
- **Тесты:** встроенный `node:test`.
- **Авторитативный сервер:** клиент шлёт только намерения, сервер бросает кубики через `crypto.randomInt` и валидирует все ходы. Анти-чит «из коробки».

## Структура

```
server/
  index.js          # Express static + WebSocket upgrade
  room.js           # GameRoom: FSM ходов, broadcast
  protocol.js       # типы сообщений + валидация
  engine/
    rules.js        # константы
    scoring.js      # scoreSelection(dice) → {score, valid, breakdown}
    validate.js     # canHold, canBank, checkWin, isFarkle
public/
  index.html        # лобби + игровой экран + модалки
  styles/           # main.css (layout) + dice.css (грани + tumble)
  js/
    main.js         # bootstrap, hotkeys
    net.js          # WebSocket + reconnect backoff
    state.js        # snapshot store + selection
    ui.js           # рендер DOM по снапшоту
    engine/         # зеркало серверного движка (preview-скор)
tests/
  scoring.test.js   # 40+ кейсов
  validate.test.js
```

## Не реализовано (есть в KCD2)

- **Специальные кости** (например, «Чёртова голова» — джокер). Используются только стандартные кости 1..6.
- **Бляхи** (одноразовые предметы вроде «Серебряная бляха удвоения»).
- **Подбор разных костей перед игрой.**
- **Ставки и валюта.**

Эти фичи могут быть добавлены позже — движок и протокол к этому готовы.
